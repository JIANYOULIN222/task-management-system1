# 產品需求文件 (Product Requirements Document)

## 1. 產品概述
本專案為一個個人任務管理系統 (Task Management System)，旨在幫助個人用戶有效率地管理日常工作與待辦事項。

## 2. 目標用戶
需要管理日常工作、追蹤待辦事項進度、提升個人生產力的個人用戶。

## 3. 系統描述
一個簡潔且具備現代感設計的網頁應用程式，讓使用者可以輕鬆地新增、完成、刪除待辦任務，並能透過狀態篩選功能快速檢視目前的任務清單。

## 4. 主要功能需求
1. **新增任務**：使用者可以輸入任務名稱並將其加入到待辦清單中。若未輸入內容則無法新增。
2. **標記任務完成**：使用者可以點擊任務旁的核取方塊，將清單上的任務標記為「已完成」，系統應在視覺上（例如：加上刪除線、文字顏色變淡）做出區隔。使用者也能將已完成的任務再次點擊取消標記，恢復為「進行中」狀態。
3. **刪除任務**：使用者可以點擊刪除按鈕，將不再需要或已作廢的任務從清單中移除。
4. **顯示所有任務清單**：系統預設應在主畫面上展示所有的任務列表。
5. **依完成狀態篩選**：提供篩選按鈕或頁籤功能，讓使用者可以快速切換檢視「全部 (All)」、「進行中 (Active)」或「已完成 (Completed)」的任務。

## 5. 非功能需求與技術規格
- **使用者介面 (UI/UX)**：採用現代化的設計風格（如：玻璃擬物化 Glassmorphism），搭配流暢的微動畫 (Micro-animations) 與高質感的色彩配置，提供極佳的使用體驗。
- **資料儲存**：使用瀏覽器的 `localStorage` 來儲存任務資料。這能確保使用者在重新整理頁面或下次開啟瀏覽器時，任務資料依然保留著。
- **響應式設計 (Responsive Web Design)**：介面需適應桌上型電腦、平板與智慧型手機的螢幕尺寸，確保在所有設備上皆能正常操作。
- **技術架構**：前端採用純 HTML5, CSS3, 與 Vanilla JavaScript (ES6+) 開發，以確保系統輕量化、載入速度快且易於維護。
